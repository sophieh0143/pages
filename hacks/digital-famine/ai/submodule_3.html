üíª JavaScript UI Functionality

UI functionality refers to the JavaScript code that makes your web page interactive. Let's explore the three main components:

1. Modal System (Pop-up Window)

A modal is a pop-up window that appears over the main page for creating and editing posts.

function openModal({ mode, post = {} }) {
  document.getElementById('microblog-modal').classList.remove('hidden');
  document.getElementById('modal-title').textContent = 
    mode === 'edit' ? 'Edit Microblog Post' : 'Create Microblog Post';
  document.getElementById('post-id').value = post.id || '';
  document.getElementById('content').value = post.content || '';
}

What this does:

  • ‚Ä¢ Makes the modal visible by removing the 'hidden' CSS class
  • ‚Ä¢ Changes the title based on create/edit mode
  • ‚Ä¢ Pre-fills form with existing data (edit) or leaves empty (create)

2. Form Submission (Saving Data)

When a user fills out the form and clicks "Save":

document.getElementById('microblog-form').onsubmit = async function(e) {
  e.preventDefault();
  
  const id = document.getElementById('post-id').value;
  const content = document.getElementById('content').value;
  
  try {
    if (id) {
      await updatePost({ id, content });
    } else {
      await createPost({ content });
    }
    closeModal();
    renderMicroblogTable();
  } catch (err) {
    alert('Error saving post: ' + err.message);
  }
};

Step-by-step:

  • 1. e.preventDefault() stops page refresh
  • 2. Collect data from form fields
  • 3. Check for ID - if exists, update; if not, create
  • 4. Call appropriate API function
  • 5. Close modal and refresh table display

3. Table Rendering (Displaying Data)

This function fetches post data and displays it in a table:

async function renderMicroblogTable() {
  const container = document.getElementById('microblog-playground');
  
  try {
    // Fetch data from API
    const data = await fetchPosts();
    
    // Build table HTML
    let tableHTML = '<table id="microblog-table">...';
    data.microblogs.forEach(post => {
      tableHTML += `<tr>
        <td>${post.userName}</td>
        <td>${post.content}</td>
      </tr>`;
    });
    tableHTML += '</table>';
    
    // Display table
    container.innerHTML = tableHTML;
    
    // Add interactive features
    $('#microblog-table').DataTable();
  } catch (error) {
    container.innerHTML = `<div style="color:red;">Error: ${error.message}</div>`;
  }
}

Why this matters: The table rebuilds from scratch every time data changes, ensuring the UI always matches current data.
